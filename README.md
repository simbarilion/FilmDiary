# 🎬 FilmDiary — персональный кино-трекер с системой рекомендаций

Адрес сервера с развернутым приложением: https://diploma.creepysnakes.su/

Fullstack приложение для ведения структурированного дневника просмотров и оценки фильмов. 
Фокусируется на персональном пользовательском опыте, без социальной составляющей.

Представляет собой оптимальную комбинацию кинодневника, персональных рекомендаций и календарного планирования.

Проект включает расширенный функционал: API интеграция для ежедневного обновления рекомендаций 
и получения расширенной информации о фильмах, Telegram-уведомления, Celery-задачи, активацию аккаунта по email 
и строгую систему прав доступа.

### Особенности проекта:
- данные пользователей полностью изолированы;
- учёт просмотренного (без привязки к предпочтениям других пользователей);
- личные отзывы (личная система хранения отзывов без необходимости публикации);
- личные оценки являются приоритетными над внешними рейтингами: в системе приоритетов личная оценка фильма 
стоит выше общего рейтинга фильмов международных рейтинговых систем;
- календарь просмотров;
- интерпретируемая рекомендательная система оценки фильмов (включает анализ поведения пользователя 
и характеристик контента).

**смотрю → оцениваю → получаю рекомендации → планирую следующее**

### Архитектурные принципы

- разделение доменной логики и API слоя;
- сервисный слой для рекомендаций; 
- кэширование внешних запросов;

````

````

````
DEBUG=False

DB_NAME=your_database_name_here

DB_PASSWORD=your_database_password_here
DB_HOST=db
DB_PORT=5432

TMDB_API_KEY=your_TMDB_api_key

EMAIL_HOST=smtp.yandex.ru
EMAIL_PORT=465
EMAIL_USE_TLS=False
EMAIL_USE_SSL=True
EMAIL_HOST_USER=your_email_host_user_here
EMAIL_HOST_PASSWORD=your_email_host_password_here

LOCATION=redis://redis:6379/1
CELERY_BROKER_URL=redis://redis:6379/2
CELERY_RESULT_BACKEND=redis://redis:6379/3

TELEGRAM_TOKEN=telegtam_token

SUPERUSER_EMAIL=superuser_email_here
SUPERUSER_PASSWORD=superuser_password_here
MANAGER_EMAIL=manager_email_here
MANAGER_PASSWORD=manager_password_here

DJANGO_LOG_LEVEL=INFO
````
4. Выполните миграции
````
python manage.py migrate
````
5. Создайте группу Manager и профили суперпользователя и менеджера через кастомную команду
````
python manage.py bootstrap_project
````
6. Запустите сервер
````
python manage.py runserver
````
7. Запустите Celery, celery-beat
````
celery -A config worker -l INFO (для Windows - celery -A config worker -l INFO -P solo)
celery -A config beat -l INFO
````

## 🧱 Архитектура проекта

### Модели
- **CustomUser**: учётная запись пользователя (email‑логин, аватар, часовой пояс, блокировка);

- **Film**: фильм - карточка фильма из TMDB, общая для всех пользователей (основная информация, жанры, 
актёры, съёмочная группа);

- **Actor / Genre / Person**: справочники актёров, жанров и участников съёмочной группы;

- **FilmActor / FilmCrew**: промежуточные связи «фильм–актёр» (с ролью и порядком) и «фильм–персона» 
(с должностью - режиссер, сценарист, продюссер);

- **UserFilm**: связь «пользователь–фильм», пользовательский опыт (избранное, дата добавления);

- **Review** – отзыв пользователя на фильм с детализацией оценок по критериям (сюжет, актеры, режиссура, 
визуал, саундтрек) и вычисляемым средним рейтингом;

- **CalendarEvent** – запланированный просмотр фильма пользователем с датой и напоминанием;

- **MessageFeedback** – сообщения обратной связи от пользователей, в том числе неавторизованных.

| Модель          | Связи и роль в системе                                                                                 |
| --------------- |--------------------------------------------------------------------------------------------------------|
| CustomUser      | Связана с UserFilm, Review, CalendarEvent, MessageFeedback                                             |
| Film            | Связана с Genre, Actor, Person (через FilmActor / FilmCrew), а также с UserFilm, Review, CalendarEvent |
| Genre           | ManyToMany с Film                                                                                      |
| Actor           | ManyToMany с Film через FilmActor                                                                      |
| Person          | ManyToMany с Film через FilmCrew                                                                       |
| FilmActor       | Film ↔ Actor с ролью и порядком                                                                        |
| FilmCrew        | Film ↔ Person с должностью                                                                             |
| UserFilm        | CustomUser ↔ Film, хранит пользовательские флаги                                                       |
| Review          | CustomUser ↔ Film, хранит оценки и текст отзыва                                                        |
| CalendarEvent   | CustomUser ↔ Film, хранит планируемый просмотр                                                         |
| MessageFeedback | Связана только с CustomUser по email (логически, не FK)                                                |

### Права доступа

- **Неавторизованный пользователь**: может просматривать Главную страницу приложения и страницу Обратной связи 
для пользователей, 
отправлять сообщения администратору через форму обратной связи;
- **Авторизованный пользователь**: имеет доступ ко всем пользовательским разделам (каталог, поиск, «мои фильмы», 
отзывы, календарь), 
может создавать, редактировать и удалять только свои объекты (UserFilm, Review, CalendarEvent, профиль) 
Попытка изменить данные другого пользователя приводит к ошибке доступа;
- **Суперпользователь**: имеет полный доступ ко всем разделам и объектам: видит и редактирует данные пользователей, 
управляет пользователями, правами и настройками приложения;
- **Менеджер**: пользователь в группе Manager. Может просматривать данные пользователей (фильмы, отзывы, календарь) 
и использовать специальные права (блокировка пользователей), но не управляет системными настройками проекта;
- **Заблокированный пользователь**: имеет доступ к странице авторизации (для показа сообщений об ограничении), 
но не может выполнять действия, изменяющие данные (добавление/редактирование/удаление фильмов, отзывов, событий, 
изменение профиля).

**Менеджер-панель**

Менеджер-панель — это административный интерфейс для управления пользователями системы.
Доступ к панели имеют только пользователи с ролью Manager и суперпользователи.

*Функциональность панели*:

- просмотр списка активных и заблокированных пользователей; 
- просмотр профиля пользователя;
- просмотр фильмов пользователя; 
- просмотр отзывов; 
- просмотр запланированных событий в календаре; 
- блокировка и разблокировка аккаунтов. 

Панель позволяет менеджерам модерировать контент и оперативно реагировать на нарушения правил.

**Блокировка пользователя**

При блокировке пользователя:
- пользователь помечается как is_blocked = True;
- все активные сессии пользователя принудительно удаляются;
- пользователь разлогинивается на всех устройствах;
- повторный вход в систему становится невозможен.

Менеджер не может заблокировать самого себя.

Повторная блокировка уже заблокированного пользователя предотвращается.

Заблокированный пользователь сохраняет возможность зайти на главную страницу приложения 
(для неавторизованного пользователя), перейти в раздел Обратня связь и отправить сообщение, заполнив форму
обратной связи. Сообщение сохраняется в базу данных и становится доступным менеджеру и суперпользователю 
для рассмотрения.

Разблокировка пользователя снимает ограничение и возвращает возможность входа в систему.

### DRF: calendar_events

- **CRUD API** (`/api/calendar_events/`) — добавление/редактирование/удаление планов просмотров
- **Два режима**: активные планы (≥ сегодня) + архив (прошедшие)
- **Автоматические напоминания** в Telegram в **12:00–12:30** по местному времени пользователя
- **Frontend** (`calendar_list.html`) — **AJAX-загрузка** с пагинацией (12/стр), группировкой по датам, 
модальными окнами
- **Валидация**: один фильм = одна дата на пользователя

#### **Техническая реализация**
    CalendarEventViewSet (DRF ModelViewSet)
    ├── queryset: фильтр по user + active/archive
    ├── ManagerOrOwnerPermission (superuser/manager + владелец)
    ├── CalendarEventSerializer (film_title, film_tmdb_id)
    ├── CalendarEventPaginator (12/стр)
    ├── upcoming (48ч вперед)
    └── AJAX Frontend (fetch + пагинация + модальные окна)
    └── Celery @daily_reminders (TG бот)
    └── Интегрировано с профилем пользователя (Telegram ID)

#### **Эндпоинты**
GET /api/calendar_events/?view=active → активные планы

GET /api/calendar_events/?view=archive → архив

GET /api/calendar_events/upcoming/ → ближайшие 48ч

POST /api/calendar_events/ → создать план

DELETE /api/calendar_events/{id}/ → отменить просмотр

### **TMDB API интеграция**

**Архитектура**

    Tmdb() → requests + Django cache (TTL 1-12ч) → TmdbFilm dataclass
     ├── _get() — GET + retry(3) + backoff(1-4s) + MD5 cache_key
     └── _get_multipage() — пагинация (до N страниц)

**Ключевые методы**

| Метод                    | Использование                                                 | Кэш TTL |
| ------------------------ |---------------------------------------------------------------| ------- |
| get_candidate_pool(1200) | Рекомендации: popular(3) + top_rated(3) + trending + upcoming | 12ч     |
| get_movie_details(id)    | Карточка фильма: overview, images, tagline                    | detail  |
| get_credits(id)          | Актёры + режиссёр (cast[:5], crew=Director)                   | credits |
| search_movie(query)      | Поиск на главной странице приложения                          | search  |
| get_movies_by_genre(id)  | Рекомендации по жанрам (топ-3 профиль)                        | genres  |

**Оптимизации**

- Cache key: MD5(path + params) — уникальный для каждого запроса
- Retry: 429/5xx → 3 попытки (1s, 2s, 4s backoff)
- Timeout: 5s защита
- Дедупликация: {tmdb_id: raw_data} в пуле кандидатов
- Русский язык: language="ru-RU"
- Постеры: get_poster_url(path, "w342")


### **Система рекомендаций**

Проект реализует персональные рекомендации на основе content-based filtering.
Система анализирует оценки пользователя и строит профиль предпочтений на основе атрибутов фильмов.

Каждый фильм представляется как набор признаков.
```
Пример:
genre:action
director:nolan
actor:leo_dicaprio
keyword:dream
```
Эти признаки формируют вектор фильма в пространстве признаков.

**Архитектура**

Рекомендательная система состоит из **трёх уровней**:

- Feature extraction — извлечение признаков фильма (жанры, режиссёр, актёры, ключевые слова);
- User profile — агрегированный профиль пользователя строится на основе просмотренных и оценённых фильмов;
- Similarity scoring — вычисление сходства между профилем пользователя и кандидатами.

**Метрики сходства**

Используется комбинированная модель:

1. Структурное сходство (70%)
 - взвешенный Jaccard по признакам: director > actor > genre > keyword
Это отражает приоритет режиссёра над жанрами.

2. Текстовое сходство (30%)
- TF-IDF косинус по:
  - overview
  - tagline
Позволяет учитывать сюжетное сходство.

3. Жанровый профиль пользователя
- дополнительный бонус кандидатам, совпадающим с топ-жанрами пользователя.

**Веса (настраиваемые)**

| Компонент | Вес | Описание                    |
| --------- | --- |-----------------------------|
| director  | 3.0 | Режиссер: самый важный      |
| actor     | 2.0 | Актер: второй по важности   |
| genre     | 1.5 | Жанр: базовый               |
| keyword   | 1.0 | Ключевое слово: минимальный |

**Оптимизации производительности**

- инвертированный индекс признаков;
- top-K ограничение кандидатов; 
- TF-IDF sparse matrix;
- FeatureCache в памяти; 
- логарифмическое затухание старых просмотров;
- кэш рекомендаций на 24 часа.

### Celery задачи
1. **send_activation_email + send_confirm_email**

Email рассылка:
- Отправляет ссылку активации при регистрации нового пользователя;
- Отправляет ссылку для подтверждения смены email;
- Реализовано через отложенную Celery-задачу.

2. **recompute_user_recommendations + recompute_all_recommendations**
- Ежедневно запускается в 01:00 UTC;
- Получает данные через API TMDB;
- Формирует рекомендации для каждого user;
- Кэширует на 24 ч.

3. **send_daily_reminders + send_telegram_message**
- Запускается каждый час;
- Находит события planned_date=сегодня + reminder_sent=False;
- Конвертирует UTC → локальная TZ пользователя -> ежедневно в 12:00–12:30 local TZ;
- Отправляет сообщение через Телеграм-бота (requests.post к Bot API);
- Помечает reminder_sent=True (один раз).

### Интеграция с Telegram
**Проект отправляет сообщения через Telegram Bot API:**
- уведомление о запланированном на текущий день просмотре: в 12:00 согласно таймзоне пользователя;
- пользователь получает уведомление один раз: при успешной отправке уведомление не дублируется.
````
Пример сообщения:

Привет!
Сегодня запланированы просмотры:
• Убежище
• Быстрее пули
````

### CI/CD (GitHub Actions)

Workflow расположен в .github/workflows/ci_cd.yaml

**Алгоритм workflow:**
- Lint
    - запуск линтеров

````
lint:
	isort . --check-only --diff
	black . --check
	flake8 .

format:
	isort .
	black .
````

- Tests

    - запуск Django-тестов

    - при ошибках деплой останавливается


- Build

    - сборка Docker-образа

    - push в Docker Hub


- Deploy

    - генерация .env

    - подмена server_name в nginx.conf

    - деплой на сервер через rsync

    - Cleanup .env из GitHub Actions runner

    - загрузка образов сервисов, указанных в docker-compose.yaml (docker compose pull)

    - запуск сервисов в фоновом режиме (docker compose up -d)

Pre-commit с линтерами запускается при каждом commit.

Workflow запускается автоматически при каждом push.

### Деплой приложения

Деплой происходит автоматически после успешного прохождения тестов.

Ручной деплой на сервере:
````
cd ~/filmdiary
docker compose pull
docker compose up -d
````
